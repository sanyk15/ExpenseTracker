# ExpenseTracker 💰💙

Мобильное приложение для отслеживания доходов и расходов с удобной статистикой и резервным копированием данных.

## ✨ Возможности

### 📊 Управление расходами

- Добавление, редактирование и удаление расходов
- Организация по категориям (иконка + цвет)
- Комментарий к каждому расходу (опционально)
- Выбор даты расхода (через календарь)
- Просмотр расходов по выбранному дню
- Быстрый отчёт по дню с общей суммой и копированием списка в буфер обмена

### 💵 Управление доходами

- Добавление, редактирование и удаление доходов
- Комментарий/описание источника дохода
- История доходов по датам
- Просмотр доходов за выбранный день

### 📈 Статистика и анализ

- Общая статистика расходов по категориям
- Статистика доходов по источникам
- Периоды:
  - Текущая неделя
  - Текущий месяц
  - Текущий год
  - Пользовательский период с выбором дат
- Учёт крайних дат: все расходы и доходы с начала первого дня до конца последнего включительно
- Детальный экран категории:
  - Сумма расходов за выбранный период
  - Список всех трат по этой категории в выбранном периоде
  - Период фильтра наследуется из общей статистики
- Графики:
  - Бар‑чарт по месяцам/периодам (расходы, доходы)
  - Наглядные прогресс‑бары по категориям и источникам

### 🏷️ Категории

- Создание собственных категорий
- Выбор эмодзи‑иконки
- Цвет категории (hex)
- Редактирование и удаление категорий
- Обновление категорий во всех связанных расходах

### ⚙️ Настройки

- Переключение тёмной/светлой темы (сохранение в UserDefaults)
- Статистика по количеству данных:
  - Сколько расходов
  - Сколько доходов
  - Сколько категорий

### 💾 Резервное копирование данных

- Экспорт всех данных в JSON:
  - Категории (название, цвет, иконка)
  - Расходы
  - Доходы
- Импорт данных из JSON:
  - Полное восстановление категорий, расходов и доходов
  - Поддержка актуального формата бэкапа
- Файл резервной копии:
  - Имя вида: `ExpenseTracker_YYYY-MM-DD_HH-mm.json`
  - Подходит для хранения в iCloud, почте, мессенджерах и т.д.
- Работа через стандартный Share Sheet и файловый диалог iOS

## 🛠️ Технологический стек

- **Язык:** Swift
- **Фреймворк:** SwiftUI
- **Хранилище:** UserDefaults (локально на устройстве)
- **Архитектура:** MVVM (`ExpenseViewModel` + отдельные `View` и `Model`)
- **Минимальная iOS:** 14.0+

## 📁 Основные экраны

| Вкладка       | Описание                                      |
| ------------- | --------------------------------------------- |
| 📊 Расходы    | Список трат за выбранный день + итоги         |
| 💰 Доходы     | Список доходов за выбранный день              |
| 📈 Статистика | Аналитика по расходам и доходам за период     |
| 🏷️ Категории | Управление категориями расходов                |
| ⚙️ Настройки  | Тема, резервные копии, счётчики данных        |

## 🚀 Запуск проекта

### Требования

- Xcode 14+
- iOS 14.0+
- Swift 5.7+

### Установка
1. Клонируй репозиторий:
```bash
git clone https://github.com/sanyk15/ExpenseTracker.git
```
2. Открой проект:
```
ExpenseTracker.xcodeproj
```
3. Выбери симулятор или подключённое устройство.
4. Нажми Cmd + R для запуска.

### 🧭 Как пользоваться
#### Добавить расход
1. Перейди на вкладку «Расходы».
2. Выбери дату в верхней части экрана.
3. Нажми кнопку "+" в правом верхнем углу.
4. Заполни сумму, категорию, дату и комментарий (опционально).
5. Сохрани.

#### Добавить доход
1. Перейди на вкладку «Доходы».
2. Выбери дату.
3. Нажми "+".
4. Заполни сумму и при необходимости комментарий.
5. Сохрани.

#### Посмотреть статистику
1. Открой вкладку «Статистика».
2. Выбери период (неделя / месяц / год / пользовательский).
3. При выборе «Пользовательский» задай диапазон дат.
4. Посмотри:
    - общую сумму,
    - разбивку по категориям (расходы),
    - разбивку по источникам (доходы).
5. Нажми на категорию, чтобы открыть детальную страницу с тратами по этой категории в том же периоде.

#### Резервная копия (экспорт)
1. Открой «Настройки» (иконка шестерёнки на экране расходов).
2. В разделе «Резервное копирование» нажми «Экспортировать данные».
3. Выбери, куда сохранить/отправить JSON файл.

#### Восстановление (импорт)
1. Открой «Настройки».
2. Нажми «Импортировать данные».
3. Выбери ранее сохранённый JSON файл.
4. Данные будут загружены и заменят текущие.
